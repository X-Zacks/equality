# Design: 设置页「模型」Tab 重设计

## 1. 整体布局

模型 Tab 分为三个垂直区块，区块之间用分隔线隔开：

```
┌──────────────────────────────────────────────────┐
│ 区块 A：当前模型（固定高度，不受下方影响）          │
├──────────────────────────────────────────────────┤
│ 区块 B：Provider 列表（固定行高，不展开折叠）       │
├──────────────────────────────────────────────────┤
│ 区块 C：网络设置                                  │
└──────────────────────────────────────────────────┘
```

点击 Provider 行的操作按钮后，**右侧滑出抽屉面板**覆盖在内容上方，
主列表保持不动。

---

## 2. 区块 A：当前模型

```
🤖 当前模型
┌────────────────────────────────────────────┐
│  MiniMax M2.7                          ▾   │   ← 模型下拉（现有组件保留）
└────────────────────────────────────────────┘
□ Auto（根据问题复杂度自动选择模型）
  所有消息将使用上方选定的模型
```

- 模型下拉和 Auto 复选框保持现有逻辑，样式不变
- 区块高度固定，任何操作都不影响它

---

## 3. 区块 B：Provider 列表

每个 Provider 一行，行高固定（48px），包含：

```
[图标] [名称]          [状态标签]        [操作按钮]
```

### 行状态视觉规范

| 状态 | 左侧装饰 | 状态标签 | 标签颜色 |
|------|---------|---------|---------|
| 激活中 | 2px 蓝色左边框 | `● 激活中` | 蓝色 `#0a84ff` |
| 已配置 | 无 | `● 已配置` | 绿色 `#30d158` |
| 未配置 | 无 | `○ 未配置` | 灰色 `#666` |
| Copilot 未登录 | 无 | `○ 未登录` | 灰色 `#666` |

### 操作按钮规范

| 状态 | 按钮文字 |
|------|---------|
| 已配置/激活 | `管理` |
| 未配置（普通） | `配置` |
| Copilot 未登录 | `登录 GitHub` |

### 完整示例

```
┌──────────────────────────────────────────────────┐
│▌ 🟣 MiniMax       ● 激活中              [管理]   │  ← 蓝色左边框
│  🐙 Copilot       ● 已登录              [管理]   │
│  🔧 自定义端点     ● 已配置              [管理]   │
│  🔵 DeepSeek      ○ 未配置              [配置]   │
│  🟡 Qwen          ○ 未配置              [配置]   │
│  🟠 Volc          ○ 未配置              [配置]   │
│  🔴 MiniMax       ○ 未配置              [配置]   │
└──────────────────────────────────────────────────┘
```

---

## 4. 抽屉面板（Drawer）

点击操作按钮后，从设置页**右侧边缘滑入**，宽度 320px，
覆盖在主内容上方，背景半透明遮罩。

### 布局

```
┌────────────────────────────────┐
│ [Provider图标] MiniMax   [×]   │  ← 标题栏
├────────────────────────────────┤
│                                │
│ API Key                        │
│ ┌──────────────────────────┐   │
│ │ eyJ****                  │   │  ← 密码输入框，可切换显示
│ └──────────────────────────┘   │
│ 获取 API Key →（外部链接）       │
│                                │
│ （Copilot 显示登录流程，        │
│   其他 Provider 显示 API Key） │
│                                │
├────────────────────────────────┤
│ [清除配置]          [保存]     │  ← 底部操作栏
└────────────────────────────────┘
```

### 交互规范

- 点击遮罩或 `×` 关闭抽屉，不保存
- 点击 `保存` 后关闭，并更新列表行状态
- 抽屉打开时主列表不可点击（遮罩阻挡）
- 动画：`transform: translateX(100%)` → `translateX(0)`，duration 200ms

### 各 Provider 抽屉内容差异

| Provider | 抽屉内容 |
|---------|---------|
| 普通（DeepSeek/Qwen/Volc/MiniMax） | API Key 输入框 + 获取链接 |
| 自定义端点 | API Key + Base URL + Model 三个输入框 |
| Copilot 未登录 | Device Flow 登录流程（现有逻辑迁移） |
| Copilot 已登录 | 已登录状态 + 退出按钮 |

---

## 5. 区块 C：网络设置

```
── 网络 ──────────────────────────────────────────
  🌐 HTTP 代理        未设置               [配置]
```

同样使用抽屉模式，点击 `[配置]` 滑出代理设置抽屉。

---

## 6. 不变部分

- 模型下拉组件（`ModelRoutingCard`）逻辑和样式保留
- Auto 模式切换逻辑保留
- 所有 `saveApiKey` / `deleteKey` / Copilot 登录流程的业务逻辑保留，只是 UI 容器改变
- 其他 Tab（工具/Skills/高级/关于）不受影响
