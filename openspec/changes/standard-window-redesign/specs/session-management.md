# Design: 会话管理交互

## 概述

用户需要能够：创建新对话、在多个对话之间切换、查看历史对话列表、删除不需要的对话。

## 交互模型

### 核心概念

- **会话 (Session)**：一次独立的对话上下文，有唯一 key、消息历史、创建时间
- **活跃会话**：当前正在交互的会话，输入框发送的消息归属于此会话
- **会话列表**：所有持久化的会话，按最后活跃时间倒排

### 布局方案

在现有侧边栏（48px 图标栏）和主内容区之间，增加一个**可折叠的会话列表面板**：

```
┌────┬──────────────┬──────────────────────────────┐
│    │ [+ 新对话]   │  顶栏（可选：当前会话标题）     │
│ 💬 │──────────────│──────────────────────────────│
│    │ ▸ 今天       │                              │
│ ⚙️ │  ● 帮我写代码│   消息列表                    │
│    │    读取文件.. │                              │
│    │ ▸ 昨天       │   AI 回复...                  │
│    │   创建 TODO  │   用户消息...                  │
│    │   分析日志.. │                              │
│    │              │──────────────────────────────│
│    │              │  [输入框...          ] [发送]  │
├────┴──────────────┴──────────────────────────────┤
│  ● Core 在线 | copilot (gpt-4o)                  │
└──────────────────────────────────────────────────┘
```

### 会话列表面板

| 属性 | 值 |
|------|-----|
| 宽度 | 220px |
| 可折叠 | 是，点击侧边栏 💬 按钮 toggle |
| 默认状态 | 展开 |
| 背景色 | 比主内容区略深 (`#161617`) |
| 分组 | 按日期：今天 / 昨天 / 最近 7 天 / 更早 |

### 会话列表项

每项显示：
- **标题**：取第一条用户消息的前 30 个字符，无消息则显示"新对话"
- **副标题**：最后一条消息的时间（相对时间：刚刚 / 3分钟前 / 昨天 14:30）
- **选中态**：左侧蓝色竖线 + 背景高亮
- **hover**：显示删除按钮（🗑）

### 交互流程

#### 1. 新建对话

- **触发**：点击面板顶部 `[+ 新对话]` 按钮
- **行为**：
  1. 生成新 sessionKey
  2. 清空消息列表
  3. 会话列表顶部出现新项（标题"新对话"）
  4. 输入框自动聚焦
  5. 首条用户消息发出后，列表项标题更新为该消息摘要

#### 2. 切换对话

- **触发**：点击会话列表中某一项
- **行为**：
  1. 保存当前会话状态（已自动 persist）
  2. 切换 sessionKey
  3. 从 Core 加载目标会话的历史消息
  4. 渲染消息列表
  5. 滚动到底部
- **限制**：streaming 中不可切换（按钮 disabled）

#### 3. 删除对话

- **触发**：hover 会话项 → 点击 🗑 按钮
- **行为**：
  1. 二次确认（可选，MVP 可省略）
  2. 调 `DELETE /sessions/:key`
  3. 从列表中移除
  4. 如果删的是当前活跃会话 → 自动切到列表第一个，或创建新对话

#### 4. 折叠/展开面板

- **触发**：点击侧边栏 💬 按钮
- **行为**：toggle 会话列表面板的显示/隐藏
- **折叠时**：主内容区占满宽度
- **状态持久化**：`localStorage('equality-panel-open')`

#### 5. 应用启动

- 从 `localStorage` 读取上次的 sessionKey
- 调 `GET /sessions` 加载会话列表
- 调 `GET /sessions/:key` 恢复当前会话消息
- 如果上次会话不存在（被删除）→ 创建新对话

#### 6. 快捷键

| 快捷键 | 行为 |
|--------|------|
| `Ctrl+N` | 新建对话 |
| `Ctrl+B` | 折叠/展开会话面板 |

## 数据流

```
前端 (React)                        Core 后端
┌─────────────┐                    ┌──────────────┐
│ sessionKey   │───sendMessage────▶│ /chat/stream  │
│ (useState)   │  (带 sessionKey)   │  sessionKey   │
│              │                    │  → getOrCreate│
│ sessions[]   │◀──GET /sessions───│  → persist()  │
│ (useState)   │                    │              │
│              │◀──GET /sessions/k─│  messages[]   │
│ messages[]   │                    │              │
│ (useState)   │───DELETE /sess/k──│  → deleteDisk │
└─────────────┘                    └──────────────┘

持久化：
  前端: localStorage → sessionKey, panelOpen
  后端: %APPDATA%\Equality\sessions\*.json → messages
```

## 会话标题生成

MVP 阶段用简单规则：
- 取第一条 user 消息的前 30 字符
- 去掉换行符，超出加 `…`
- 无消息时显示"新对话"

后续可以：
- 用 LLM 生成 5 字摘要标题（异步，不阻塞）
- 支持用户手动重命名

## API 补充

现有 API 已满足需求：

| API | 用途 |
|-----|------|
| `GET /sessions` | 列表（含 key, createdAt, lastActiveAt, messageCount） |
| `GET /sessions/:key` | 历史消息（过滤后只返回 user/assistant） |
| `DELETE /sessions/:key` | 删除会话 |
| `POST /chat/stream` body.sessionKey | 指定会话发消息 |
