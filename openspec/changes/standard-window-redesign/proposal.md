# Proposal: 桌面客户端从浮窗改为标准窗口

## 意图

将 Equality 桌面客户端从 **透明悬浮窗（Spotlight 风格）** 改为 **标准大窗口应用**。

当前浮窗模式在实际使用中存在严重体验问题：
1. 浮窗遮挡浏览器等其他窗口，无法拖动回避（`alwaysOnTop` + 无标题栏）
2. 跳过任务栏（`skipTaskbar`），无法通过 Alt+Tab 在窗口间切换
3. 空间受限（680×120），回复内容超过几行就需要滚动，不适合长对话

## 背景

Phase 0 设计文档中选择了 Spotlight/Raycast 风格的悬浮窗，理由是"快速呼出、轻量交互"。
但实践证明，AI 对话场景不同于启动器——回复通常较长，用户需要上下文参考，需要稳定的窗口空间。

当前设计的 Alt+Space 热键 + 系统托盘 + alwaysOnTop 悬浮窗 → 适合 Launcher，不适合 Chat。

## 做什么

1. 将 Tauri 窗口从透明无边框悬浮窗改为**标准有装饰窗口**（标题栏、最大化/最小化/关闭按钮）
2. 窗口显示在任务栏中，**支持 Alt+Tab 切换**
3. 前端改为**侧边栏导航 + 主内容区**布局：
   - 💬 对话页面（输入框在底部，对话内容在上方，充分利用大窗口空间）
   - ⚙ 设置页面（原 Settings 组件适配到主内容区）
4. 移除 Alt+Space 全局热键（标准窗口不需要）
5. 保留系统托盘（用于关闭窗口后快速恢复）
6. 窗口关闭行为改为最小化到托盘（而非退出进程）

## 不做什么（明确边界）

- ❌ 不改变 Core 后端（Gateway、Session、Provider 等完全不变）
- ❌ 不改变 Tauri Commands / proxy.rs（invoke 接口保持不变）
- ❌ 不增加多窗口支持（只有一个主窗口）
- ❌ 不添加 Markdown 渲染（后续变更）
- ❌ 不添加对话历史列表（后续变更）

## 成功标准

- [ ] 启动后出现标准窗口（带系统标题栏），在任务栏中可见
- [ ] Alt+Tab 可以正常切换到 Equality 窗口
- [ ] 侧边栏可在"对话"和"设置"之间切换
- [ ] 对话页面：底部输入框发送消息，上方区域显示回复，空间充裕
- [ ] 点击窗口关闭按钮 → 最小化到托盘（不退出）
- [ ] 托盘左键单击 → 恢复窗口
- [ ] 托盘右键 → 退出，进程完全退出
