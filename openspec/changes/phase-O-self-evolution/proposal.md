# Phase O: 自我进化闭环

> 变更名称: phase-O-self-evolution  
> 创建日期: 2026-04-09  
> 状态: 提案

---

## 动机

Equality 当前是一个**无记忆感知、无长对话适应、无跨会话学习**的 Agent。用户每次对话都在重复自己，Agent 不会从错误中学习，长对话会因 context 溢出而失忆。

参考 hermes-agent 的"自我改进闭环"设计：

```
对话 → 学习 → 记忆 → 技能化 → 历史搜索 → 更好的对话 → 更好的完成任务
```

Equality 需要补齐这个闭环中的 4 个关键环节。

## 范围

| 子阶段 | 名称 | 核心能力 |
|--------|------|---------|
| **O1** | 记忆增强 + 迭代预算 | 冻结快照保护 prompt cache + 容量管理 + 预算压力警告 |
| **O2** | 上下文压缩 | 长对话自动摘要、保护头尾、迭代更新摘要 |
| **O3** | 技能系统增强 | Agent 完成复杂任务后自动沉淀 Skill、自动触发已有 Skill |
| **O4** | 会话搜索 | 跨会话 FTS5 全文搜索 + LLM 摘要召回 |

## 设计约束

1. **Copilot 计费模型感知**: 当前 Equality 主要通过 GitHub Copilot 使用 LLM，按月订阅包含固定 premium requests（Business 300/月, Enterprise 1000/月, Pro+ 1500/月）。上下文压缩和会话搜索的 LLM 调用会消耗额外 request 配额——**使用当前对话同模型进行压缩**，不引入额外的廉价模型依赖。
2. **渐进式交付**: O1→O2→O3→O4 可逐步上线，每个子阶段独立有验证价值。
3. **向后兼容**: 无 Agent 绑定的旧会话继续正常工作，零迁移成本。
4. **Prompt cache 友好**: 借鉴 hermes-agent 的冻结快照模式，会话期间 system prompt 中的记忆内容不变。

## 非目标

- 多平台网关（Equality 是桌面应用，非目标）
- Memory Provider 插件化（当前阶段不需要第三方记忆后端）
- 并行工具执行（可作为后续优化，本 Phase 不涉及）

## 成功指标

1. 长对话（>30 轮）不再因 context 溢出报错
2. Agent 在新会话中能回忆上个会话的用户偏好和决策
3. 复杂任务完成后 Agent 主动提议沉淀 Skill（已有基础，需增强触发）
4. 用户可通过 session_search 工具搜索历史对话
5. 迭代预算到达 70%/90% 时 LLM 收到明确提醒
