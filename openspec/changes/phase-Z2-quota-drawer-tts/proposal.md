# Proposal: Phase Z2 — 配额设置移入抽屉 + 语音播报

## 背景

1. **配额设置位置不合理**：Phase Z 将配额编辑表单放在 Settings 页底部独立区域，与模型无关联感。更合理的位置是在每个 Provider 的管理抽屉内，让用户在管理具体模型时直接设置该模型的配额。
2. **语音播报缺失**：Phase Z 添加了语音输入（STT），但没有语音输出（TTS）。办公场景下，AI 回复的语音播报是自然交互的另一半。

## 目标

- Z2.1: 将配额表单从 Settings 底部移入 ProviderDrawer 抽屉组件内
- Z2.2: 在 Chat 界面添加语音播报（TTS），助手消息回复完成后可朗读

## 范围

| 变更 | 文件 | 影响 |
|------|------|------|
| 抽屉内配额表单 | `Settings.tsx` — ProviderDrawer | 中等（移动 + 重构） |
| 移除底部配额区 | `Settings.tsx` — 模型Tab底部 | 小（删除） |
| 语音播报逻辑 | `Chat.tsx` | 中等 |
| 播报按钮样式 | `Chat.css` | 小 |

## 非目标

- 第三方 TTS 服务（先用浏览器原生 SpeechSynthesis API）
- 流式播报（等完整回复后再播报，避免截断感）
