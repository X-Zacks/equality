# Proposal: Tool Loop Detection

> Phase 6 | 优先级: 🔴 P0  
> 对标: OpenClaw tools context guard

## 意图

Agent 在工具调用循环中可能陷入死循环——反复以相同参数调用同一工具、或两个工具之间来回乒乓——消耗大量 tokens 却无产出。当前系统仅有一个粗暴的 `MAX_TOOL_CALLS = 30` 计数器，无法区分正常的高频工具使用和真正的循环。

## 目标

实现 4 个分级检测器，精确识别不同类型的循环模式，在警告和终止之间提供缓冲：

1. **generic_repeat** — 同参数同结果连续重复（最通用）
2. **poll_no_progress** — 轮询类工具（bash/process）无进展（更早触发）
3. **ping_pong** — A→B→A→B 交替循环（需双方结果稳定才判定）
4. **circuit_breaker** — 全局工具调用次数兜底（最后防线）

## 范围

- **包含**：循环检测逻辑、Hash 基础设施、runner 集成
- **不包含**：用户可配置阈值（后续迭代）、UI 循环警告展示

## 成功标准

- Agent 陷入 bash 轮询循环时，10 次后收到警告，20 次后自动终止
- 乒乓循环 20 次交替后自动终止
- 正常的高频工具使用（如连续读取多个不同文件）不受影响
