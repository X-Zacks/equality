# Proposal: read_pdf_vision — 扫描件 PDF 视觉识别工具

## 动机

当前 `read_pdf` 基于 `pdf-parse` 提取文字层，对扫描件 PDF（纯图片）无能为力。
用户处理合同、报告等扫描件时，需要能还原表格和完整页面信息。

## 核心需求

1. **PDF → 逐页图片 → 视觉模型识别**：自动将 PDF 每页渲染为 PNG，调用视觉模型提取文字和表格
2. **临时文件清理**：识别完成后物理删除所有转换的 PNG 图片
3. **断点续传**：识别过程中出错时，已识别的页面结果缓存，恢复后不重复识别
4. **成本优化**：优先用 Copilot GPT-4o 完成识别（免费/低成本），无 Copilot 时降级到用户当前模型
5. **自动判断**：LLM 调用 `read_pdf` 时自动检测是否为扫描件，自动降级到 `read_pdf_vision`；用户通过 `#` 指定工具时优先使用用户选择

## 范围

- 新增 `read_pdf_vision` 工具
- 修改 `read_pdf` 添加自动降级逻辑
- 纯 JS 实现（pdfjs-dist 渲染 + canvas），无外部依赖
