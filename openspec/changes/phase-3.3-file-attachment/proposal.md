# Proposal: Phase 3.3 — 对话附件（文件选择 & 拖拽）

## 动机

用户在对话中经常需要引用本地文件：

- "帮我读一下这个 PDF"
- "分析这张图片"
- "检查这段代码有什么问题"

目前用户只能手动输入文件路径，体验很差。OpenClaw 的 image/pdf 工具都支持 URL 或本地路径传入，但用户交互层没有便捷入口。

需要在对话输入区提供**文件选择按钮**和**拖拽放置**两种方式，让用户轻松附加文件。

## 范围

### In Scope
1. 输入区左侧添加 📎 按钮，点击打开系统文件选择对话框
2. 支持拖拽文件到对话输入区
3. 已选文件以 **紧凑标签** 形式显示在输入框上方
4. 发送时将文件路径注入用户消息，LLM 自动识别并调用对应工具
5. 支持多文件附加（最多 5 个）
6. 支持的文件类型：图片、PDF、文本/代码文件

### Out of Scope
- 文件上传到服务器（我们传递本地路径，工具直接读取本地文件）
- 文件预览（图片缩略图等，后续可加）
- 云端文件 / URL 附件

## 技术路线

Tauri 桌面应用可直接获取本地文件路径（无需上传），这是相对 Web 的重要优势。

- 文件选择：使用 Tauri 的 `dialog.open()` API
- 拖拽放置：使用 HTML5 drag-and-drop + Tauri 的 `tauri://drag-drop` event
- 路径传递：附件路径注入到用户消息文本中，格式为 `[附件: /path/to/file.pdf]`
- 后端无需改动：LLM 看到路径后自然会调用 read_file / read_image / read_pdf

## 安全考虑
- 仅接受本地文件路径（不处理远程 URL）
- 单个文件大小限制由工具层控制（read_image 10MB, read_pdf 20MB）
- 最多同时附加 5 个文件
