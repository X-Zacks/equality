# 领导介绍 Equality 演示素材包

## 使用说明
本素材包用于向领导演示 Equality 在真实办公场景中的价值，适合配合 PPT 或现场操作使用。

建议演示顺序：
1. 零散内容整理成汇报提纲
2. 两组数据差异分析
3. 网页/PDF 摘要提炼

# Demo 1：零散内容整理成汇报提纲

## 原始素材
```text
这周项目推进整体还算顺利，不过有几个点需要关注。第一，销售那边已经把第一批试点客户名单给到我们了，现在一共是 12 家，里面有 4 家反馈意愿比较强，希望能尽快安排演示。第二，产品这边把之前讨论的两个功能优先级重新排了一下，决定先做基础报表导出，把高级筛选放到下一阶段，因为目前客户最关心的是结果能不能先导出来用。第三，研发说主流程已经打通了，但接口联调过程中发现老系统返回字段不太稳定，导致有两处映射要重做，预计会多花 3 到 5 天。第四，运营同事已经整理出首版培训材料目录，但还没有开始写具体内容。第五，老板上次提到希望月底前至少完成一轮真实客户试用，这个目标现在看有机会，但前提是联调问题本周内要收敛。风险方面，一个是客户那边现在期望值比较高，担心如果演示版本体验不稳定，会影响后续推进；另一个是内部资源还是偏紧，研发本周还有别的项目插入。建议下周重点盯三件事：接口联调收口、演示版本准备、培训材料先出初稿。
```

## 建议提示词
```text
下面是一段比较零散的项目进展信息，请帮我整理成适合向领导汇报的结构化提纲。

要求：
1. 提炼成 4-6 个一级要点
2. 每个要点下用 2-3 条简洁说明
3. 单独归纳“当前风险”和“下周重点”
4. 语言正式、清晰、适合管理汇报
5. 最后输出一段 100 字以内的汇报总结
6. 输出格式为 Markdown

原始内容：
【粘贴素材】
```

## 演示讲解要点
- 适合展示“从零散素材到汇报初稿”的提效能力
- 强调价值在于结构化整理，而不是简单改写
- 可用于周报、月报、会议纪要、项目阶段汇报等场景

# Demo 3：网页/PDF 摘要提炼

## 模拟文章标题
《企业数字化工具落地，为什么试点容易、推广困难？》

## 模拟正文素材
```text
很多企业在引入数字化工具时，往往会经历一个共同阶段：试点效果不错，但进入推广阶段后进展明显放缓。造成这种现象的原因，通常不在工具本身，而在组织协同、流程适配和使用门槛。

第一，试点阶段往往聚焦在少数积极团队，参与人员意愿高、配合度强，因此更容易取得局部成功；但推广阶段会面对更多真实场景，业务差异和流程复杂度迅速上升，导致原有做法难以直接复制。

第二，很多团队在试点时强调“能不能用”，而在推广时真正影响效率的，是“是否容易持续使用”。如果工具需要额外学习成本、操作步骤多，或者与现有工作流程脱节，员工即使认可价值，也未必愿意长期使用。

第三，管理层在试点阶段通常关注结果展示，但到了推广阶段，更需要建立清晰的责任分工、培训机制和反馈闭环。没有这些配套动作，工具上线后容易变成少数人使用、多数人观望的状态。

第四，推广失败还有一个常见原因，就是没有把成功经验标准化。试点成功往往依赖个别骨干推动，如果没有把关键动作沉淀成模板、流程和可复制方法，组织很难形成规模化应用。

因此，数字化工具从试点走向推广，关键不只是证明“这个工具有效”，更要证明“这个工具可以低门槛地融入日常工作，并且能被团队持续复用”。
```

## 建议提示词
```text
请阅读下面这份材料，并输出一份适合领导快速浏览的摘要。

要求：
1. 用 3-5 条总结核心观点
2. 提炼管理层最关心的信息
3. 单独输出“推广落地的关键条件”
4. 输出“对我们有什么启发”
5. 语言简洁、正式
6. 输出格式为 Markdown

材料内容：
【粘贴素材】
```

## 演示讲解要点
- 适合展示对外部资料、行业文章、政策文件、方案文档的快速提炼能力
- 强调管理层关心的是“核心观点、落地条件、启发”而不是全文细节
- 可用于资料阅读、行业研究、政策解读、方案消化等场景

# 演示前检查清单

1. 提前把素材保存到记事本或文档中，避免现场查找
2. 每个 Demo 的提示词提前保存好，便于复制粘贴
3. 每个 Demo 只突出一个核心价值点，不要讲太散
4. 结果出来后重点讲“节省了什么时间、提炼了什么重点、为什么适合管理场景”
5. 结尾一定强调“可复用、可沉淀、可推广”
